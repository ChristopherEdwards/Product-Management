Name Value
NAME XQRESTRICT-LOCK
HEADER RESPONDING TO THE 'LOCK' PROMPT
TEXT 
   An option which is locked can only be accessed by users who possess
that key. Locks are useful in restricting access to certain options
to whole groups of users.
 
   If an option has a lock, it will be displayed as such by the menu
system, whether or not the user possesses the key.  However, it is only
selectable with the key.
   
   By responding to this prompt with an '@' you will delete any
locks currently associated with the options chosen, leaving the
field null, or unlocked.
  
   In order to specify a new lock for an option or set of options
you must first create a key in the KEY FILE, if one does not already
exist, which matches exactly what you are trying to enter in this
field.
