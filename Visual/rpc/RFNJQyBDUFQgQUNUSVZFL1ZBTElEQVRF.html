DSIC CPT ACTIVE/VALIDATE (1621)

Name Value
NAMEDSIC CPT ACTIVE/VALIDATE
TAGACTIVE
ROUTINEDSICCPT
RETURN VALUE TYPESINGLE VALUE
AVAILABILITYAGREEMENT
DESCRIPTION
This will tell you whether or not a CPT code is active as of the date 
that you pass in.
INPUT PARAMETER
  1. INPUT PARAMETER:
    CODE
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    7
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
    This is the CPT, HCPCS, or level III code from file 81 in either internal 
or external format.
  2. INPUT PARAMETER:
    CDT
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    14
    REQUIRED:
    NO
    SEQUENCE NUMBER:
    2
    DESCRIPTION:
    This date is optional.  It can be <null>, TODAY, NOW, or a internal 
Fileman format date.  Default value is TODAY.  This date will be used to 
check if the code was active as of that date.
  3. INPUT PARAMETER:
    SRC
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    1
    REQUIRED:
    NO
    SEQUENCE NUMBER:
    3
    DESCRIPTION:
    This boolean flag (1 or 0) indicates whether or not Level III codes 
should be screened out.  If SCR=1 then allow Level III codes.  Default 
value is 0.
RETURN PARAMETER DESCRIPTION
On error, return -1
Else return 1 if active, 0 if inactive
