DENTV FILE PARAMETERS (1959)

Name Value
NAMEDENTV FILE PARAMETERS
TAGPAR
ROUTINEDENTVTP0
RETURN VALUE TYPESINGLE VALUE
AVAILABILITYAGREEMENT
DESCRIPTION
This RPC allows the Dental software to file word processing type 
parameters.
INPUT PARAMETER
  1. INPUT PARAMETER:
    ENT
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    80
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
    Entity is the level where the parameter is stored (USR, PKG, SYS), etc.
  2. INPUT PARAMETER:
    PAR
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    80
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    2
    DESCRIPTION:
    Parameter is the name of the parameter that values are stored under.
  3. INPUT PARAMETER:
    INST
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    80
    REQUIRED:
    NO
    SEQUENCE NUMBER:
    3
    DESCRIPTION:
    Instance is defaulted to 1 if not defined, otherwise it represents the 
next level where parameters are stored within the entity/parameter 
combination.
  4. INPUT PARAMETER:
    DATA
    PARAMETER TYPE:
    LIST
    MAXIMUM DATA LENGTH:
    250
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    4
    DESCRIPTION:
    DATA is the value of the parameter, passed as a list to allow setting of 
word processing parameter data.
RETURN PARAMETER DESCRIPTION
RET = 1^Success if parameter was filed okay, else
     -1^error message
