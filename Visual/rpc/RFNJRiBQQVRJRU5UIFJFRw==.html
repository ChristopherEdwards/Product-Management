DSIF PATIENT REG (2657)
Name Value
NAMEDSIF PATIENT REG
TAGREG
ROUTINEDSIFNOT3
RETURN VALUE TYPESINGLE VALUE
DESCRIPTION
This RPC will add entries to file 162.2 FEE NOTIFICATION/REQUEST FILE.  
The logic was taken from routine: FBCHREQ* and edit template FBCH ENTER 
REQUEST.

    DFN
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    99
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
    Patient File (#2) pointer (Required)
  2. INPUT PARAMETER:
    DATE
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    14
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    2
    DESCRIPTION:
    Date and Time (Required - FM format)
Date and Time that the vendor notifies the COJ of this episode of 
hospitalization.

    3

  4. INPUT PARAMETER:
    DTOA
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    14
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    4
    DESCRIPTION:
    Date & Time of Admission (Required - FM format)
This field contains the date and time that you begin accepting

RETURN PARAMETER DESCRIPTION
Return Values
        -1 ^ Invalid Input!
        -1 ^ This notification has a status of complete.  Cannot edit.
        -1 ^ This Authorization From Date exceeds the 72 hour notification
period.
        NNN = Internal number of newly created record
