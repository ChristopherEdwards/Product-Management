ORQQVS VISITS/APPTS (194)

ROUTINEORQQVS

DESCRIPTION
Returns a list of patient appointments and visits for a date/time range.
location.
INPUT PARAMETER
  1. INPUT PARAMETER:
    PATIENT ID
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    16
    REQUIRED:
    YES
    DESCRIPTION:
    Patient ID (DFN) from Patient File [#2].
  2. INPUT PARAMETER:
    START DATE/TIME
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    16
    REQUIRED:
    NO
    DESCRIPTION:
    Start date/time to return visits and appointments.  In internal FileMan format.
  3. INPUT PARAMETER:
    STOP DATE/TIME
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    16
    REQUIRED:
    NO
    DESCRIPTION:
    Stop date/time to return visits and appointments. In internal FileMan format.
  4. INPUT PARAMETER:
    DUMMY
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    16
    REQUIRED:
    NO
    DESCRIPTION:
    This is a dummy parameter so the Broker won't crash.
appointments for this location only will be returned.
RETURN PARAMETER DESCRIPTION
List of patient appointments and visits in the following format:
visit/appt id (inpatient admissions include an "a" suffix, visits suffix is 
"v")^visit/appt location^header of "admitted:" or "visited:"^date/time of
visit or appt^inverse date/time of visit or appt
all dates in the system will be returned.  If no input location is
indicated, visits and appointments across all locations will be returned.
