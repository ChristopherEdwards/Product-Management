DSIF CHECK OVERLAP DATES (2659)

Name Value
NAMEDSIF CHECK OVERLAP DATES
TAGCHKOVLP
ROUTINEDSIFNOT3
RETURN VALUE TYPESINGLE VALUE
DESCRIPTION
This RPC checks the from and to dates to make sure they do not overlap 
with another request.  Logic copied from FBAAUTL2.
INPUT PARAMETER
  1. INPUT PARAMETER:
    DFN
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    99
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    1
    DESCRIPTION:
    Patient File Pointer (File #2, Required)
  2. INPUT PARAMETER:
    FBBEGDT
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    14
    REQUIRED:
    YES
    SEQUENCE NUMBER:
    2
    DESCRIPTION:
    From Date to check (FM format, Required)
  3. INPUT PARAMETER:
    FBENDDT
    PARAMETER TYPE:
    LITERAL
    MAXIMUM DATA LENGTH:
    14
    REQUIRED:
    NO
    SEQUENCE NUMBER:
    3
    DESCRIPTION:
    To Date (FM format, Optional)
RETURN PARAMETER DESCRIPTION
Return Values
        -1^Invalid Input!
        -1^FROM Date entered overlaps another request for this patient.
        -1^TO Date entered overlaps another request for this patient.
        1 = Date(s) are good
